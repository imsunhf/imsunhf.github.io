RESTful

只见XX老师在黑板上写下了 RESTful 几个大字，然后标准的问道：哪位同学课前预习了来解释下这是什么意思？

一句话解释

URL定位资源，用HTTP动词（GET,POST,DELETE,DETC）描述操作。

通俗易懂解释

  • 看Url就知道要什么
  • 看http method就知道干什么
  • 看http status code就知道结果如何

权威版

当你想更正别人的仓库时，要走一个流程

  1. 先fork 别人的仓库，不会有人让你直接改原仓库的
  2. clone 到本地分支，做一些bug fix
git clone https://github.com bootstrap
cd  bootstrap
git checkout -b test-pr
git add . && git commit -m "test-pr"
git push origin test-pr
  1. 发起 pull request给原仓库，让他看到你修改的bug 点击 仓库 compare & pull request 按钮
  2. 原仓库 review 这个bug，如果正确的话，就会merge到他自己的项目中 至此 整个 pull request的过程就结束了
